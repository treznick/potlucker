/rest/users

  • GET: lists all users in JSON
  • GET {key}/{value}: gets a user by key and value
  • POST /create : creates a user, must have name as a string and pantry as a ; delineated string
  • GET /delete/{key} : deletes a user by its primary key

/rest/recipes

  • GET /{ingredients} : gets recipes by an ingredient string, which is a whitelist string separated by ;
